§ 01

Overview

This module replaces the default Odoo product configurator for products that have a BS Dynamic BoM. Instead of creating variants up front, the user configures dimensions and options in a wizard, the rules compute the components, and the variant + Manufacturing Order are created only when the Sales Order is confirmed.

1
SO Draft — User clicks "Configure Product"
action_config_start() detects a BS BoM on the selected product and opens bs.sale.config.wizard.
2
Wizard — Preview or Apply
Preview → runs all rules, shows computed components; SO unchanged.
Apply → stores config JSON on SO line; no variant or MO yet. Gear icon appears for re-configuration.
3
SO Confirmed — Variant & MO created
For each bs_is_configured line: re-evaluates rules with stored values → finds or creates product.product variant → creates mrp.production → sets line.product_id = real variant.
Wizard before parameters are entered
After parameters set — Preview shows computed components

2.3 Migrate an Existing Product (Old Configurator → BS Wizard)

  
If the product was previously configured with product_configurator_sale, use this migration pattern:

  

    

      PYTHONMigration Script
      
    

    
# 1. Remove old product.config.session records
sessions = env['product.config.session'].sudo().search([('product_tmpl_id', '=', tmpl.id)])
sessions.sudo().unlink()

# 2. Archive all existing variants
variants = tmpl.with_context(active_test=False).product_variant_ids
variants.sudo().write({'active': False})

# 3. SQL-delete FK children first, then PTALs
env.cr.execute("""
    DELETE FROM product_variant_combination
    WHERE product_template_attribute_value_id IN (
        SELECT ptav.id FROM product_template_attribute_value ptav
        JOIN product_template_attribute_line ptal ON ptal.id = ptav.attribute_line_id
        WHERE ptal.product_tmpl_id = %s
    )
""", (tmpl.id,))
env.cr.execute(
    "DELETE FROM product_template_attribute_line WHERE product_tmpl_id = %s",
    (tmpl.id,)
)

# 4. Flush ORM cache after raw SQL — avoids stale-cache duplicate-key errors
env.cache.invalidate()

# 5. Add new attributes / values and write to template

# 6. Activate and enable configurator
tmpl.sudo().write({'config_ok': True, 'active': True})
env.cr.commit()

  

  

    
▲

    

      
Critical ordering notes

      
Delete product_variant_combination before product_template_attribute_line (FK cascade order). Call env.cache.invalidate() after every raw SQL block — without it the ORM search cache returns stale results and can cause UniqueViolation on subsequent creates.

5.3 UoM Rule — Critical

  

    
▲

    

      
UoM category mismatch causes MO error

      
The uom_id in your result must be in the same UoM category as the component product's own uom_id. Mismatches cause: "You cannot perform the move because the unit of measure has a different category as the product unit of measure."

    

  

  

    

      
Component product UoMBoM line UoM to setRule uom_id

      
        
เมตร (Length)เมตรline.product_uom_id.id

        
เส้น / ชิ้น (Unit)เส้น / ชิ้นline.product_uom_id.id

        
ม้วน (Unit)ม้วนline.product_uom_id.id

      
    

  

  
Best practice: set the BoM line's UoM to match the component product's UoM, then always use line.product_uom_id.id in the rule — zero hardcoding.

  
Ceiling division without math.ceil (not available in sandbox):

  

    

      PYTHONCeiling Division
      
    

    
length_m = width / 100
pieces   = int(length_m / piece_length_m)
if length_m > pieces * piece_length_m:
    pieces += 1
if pieces < 1:
    pieces = 1

# Alternative one-liner:
pieces = int(x / d) + (1 if x % d > 0 else 0)

5.5 Example — Piece Count (เส้น UoM)

  

    

      
B

      

        
Top Box — ceiling division to piece count

        
Component is sold by เส้น (piece), each piece covers 5.8 m. Qty must be number of pieces, not meters.

      

    

    

      

        

          PYTHONRule B — กล่องบน 38มม.

side      = values.get('ดึงและปรับ', 'ขวา')

    'ดึงและปรับ': side,

# Ceiling division: pieces = ceil(width_m / 5.8)

pieces      = int(length_m / piece_len_m)

    pieces += 1
if pieces < 1:
    pieces = 1

if top_color == 'ดำ':

    box_id, box_name = 148590, 'กล่องบน 38มม. สีขาว'

        'product_id': box_id,
        'name':       box_name,
        'qty':        pieces,
        'uom_id':     line.product_uom_id.id,   # เส้น — must be set on the BoM line

]

§ 08

  
Common Errors & Fixes


  
"Missing required fields on accountable sale order line"

  

    
▲

    

      
DB constraint — product_id IS NOT NULL

      
Cause: product_id is NULL — happens when all variants are archived (e.g. after migration) and the placeholder fallback finds nothing.

      
What the code does: action_apply() has a 3-tier fallback: (1) active variant, (2) archived variant, (3) create a blank placeholder — replaced by the real variant at SO confirmation.

    

  


  
"Cannot resolve component: [name]"

  

    
▲

    

      
Resolver cannot find product.product

      
Cause: _resolve_product() cannot find a product matching product_id, default_code, or name from the rule result.

      
Fix: Confirm the product exists in the target database. Always prefer product_id (integer ID) — name matching breaks if the name differs across environments.

    

  


  
"Attribute value X is not configured on product Y"

  

    
▲

    

      
PTAV missing on template

      
Cause: product_attributes contains a value with no product.template.attribute.value record on this product template.

      
Fix: Check the product template's Variants tab — every value referenced in product_attributes must appear there. Add missing values to the attribute line, or correct the rule.

    

  


  
"Unit of measure has a different category"

  

    
▲

    

      
UoM category mismatch on MO move

      
Fix (three steps):

      
    
        
  1. Identify the component product's UoM (e.g. เส้น = Unit category)
    2. 
        
  2. Update the BoM line's UoM to match (e.g. เส้น)
    3. 
        
  3. Update the rule: use line.product_uom_id.id for uom_id, recalculate qty to be in that UoM (e.g. piece count via ceiling division)
    4. 
      

    

  


  
"BoM has no component lines"

  

    
▲

    

      
bom_line_ids is empty

      
Cause: The BoM has no lines at Preview/Apply time. Fix: Add at least one component line to the BoM.
